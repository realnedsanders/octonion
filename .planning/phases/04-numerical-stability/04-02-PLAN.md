**Section 1 -- Depth Sweep / Error Accumulation (SC-1 + SC-3):**

Function `run_depth_sweep()` returns a dict keyed by `(algebra_name, depth, magnitude_regime)`.

For EACH of the TWO experiment types (stripped chain and full AlgebraNetwork):
1. For each algebra in {R, C, H, O}:
2. For each depth in {10, 50, 100, 500}:
3. For each input magnitude regime in {near-unit (1.0), small (0.01), large (100.0)}:
   - Build layers at float64, clone weights to float32 via `.to(torch.float32)` (NOT re-initialize)
   - Generate N_SAMPLES=500 random inputs at float64 with the target magnitude
   - Forward pass through both dtype chains
   - Compute relative error: `(out32.double() - out64).norm() / (out64.norm() + 1e-30)`
   - Compute norm drift for stripped chain: track `||output||/||input||` ratio
   - Record mean, std of relative errors across samples

For the stripped chain: build `nn.ModuleList` of `ALGEBRA_LINEAR[algebra](HIDDEN, HIDDEN, bias=False, dtype=torch.float64)`.

For the full AlgebraNetwork: build via `NetworkConfig(algebra=algebra, topology="mlp", depth=depth, base_hidden=HIDDEN, activation="split_relu", output_projection="flatten", use_batchnorm=True, input_dim=HIDDEN * algebra.dim, output_dim=HIDDEN * algebra.dim)`. Warmup BN stats: run 20 forward passes in train mode with `torch.no_grad()` before switching to eval mode for measurement. Clone the entire model to float32 via `model_f32 = copy.deepcopy(model_f64).float()`.

Input generation: `x = torch.randn(batch, HIDDEN, algebra.dim, dtype=torch.float64) * magnitude` for hypercomplex (dim > 1); `x = torch.randn(batch, HIDDEN, dtype=torch.float64) * magnitude` for real.

For full AlgebraNetwork inputs: `x = torch.randn(batch, HIDDEN * algebra.dim, dtype=torch.float64) * magnitude` (flattened real-valued input matching input_dim).

**Compute "stable depth":** For each algebra and magnitude regime, find the maximum depth where mean relative error stays below `STABILITY_THRESHOLD = 1e-3`. Interpolate between measured depth points if needed (or report the last depth below threshold). This directly supports SC-3.

**Section 2 -- Condition Number Characterization (SC-2):**

Function `run_condition_numbers()` returns condition number statistics.

**Primitive operations** (octonion-only, comparing against analytic where available):
- `octonion_mul(a, .)` for fixed random `a` -- use `numeric_jacobian(lambda x: octonion_mul(a, x), x)`
- `octonion_inverse(.)` -- use `numeric_jacobian(lambda x: octonion_conj(x) / octonion_inner(x, x), x)` or import the inverse function
- `octonion_exp(.)` -- use `numeric_jacobian(octonion_exp, x)`
- `octonion_log(.)` -- use `numeric_jacobian(octonion_log, x)` (restrict to principal branch: ||Im(x)|| < pi)

For each primitive, for each magnitude in MAGNITUDES:
1. Generate 100 random octonion inputs normalized to target magnitude
2. Compute numeric Jacobian with `eps = 1e-7 * max(1.0, magnitude)` (scale eps for large inputs, per RESEARCH.md pitfall 3)
3. Compute `svdvals = torch.linalg.svdvals(J)`; condition number = `svdvals[0] / svdvals[-1].clamp(min=1e-30)`
4. Record mean, std, median, max of condition numbers

**N-layer compositions** (all 4 algebras):
For each algebra, build stripped chains of depth 2, 5, 10 at float64. Forward pass as a single function for the full chain. Compute numeric Jacobian of the full chain function at magnitude-1.0 input. Record condition numbers.

**Full network condition numbers** (all 4 algebras):
For HIDDEN=8 algebra units, build AlgebraNetwork at depth 3 with BN. Warmup BN stats (20 passes in train mode). In eval mode, compute numeric Jacobian of the full forward pass. Record condition numbers at magnitudes {0.01, 1.0, 100.0}.

Condition number computation:
```python
def compute_condition_number(fn, x, eps=1e-7):
    J = numeric_jacobian(fn, x, eps=eps)
    sv = torch.linalg.svdvals(J)
    return (sv[..., 0] / sv[..., -1].clamp(min=1e-30)).item()
```

- Stable depth per algebra (stripped chain, near-unit magnitude)
- Stable depth per algebra (full AlgebraNetwork, near-unit magnitude)
- Cross-algebra comparison table

No additional measurements needed beyond Section 1. This section organizes and presents the SC-3 specific view.

**Section 4 -- Mitigation Demonstration (SC-4):**

Function `run_mitigation()` returns before/after stable depth comparison.

For each algebra, for each K in {5, 10, 20}:
1. Build stripped chain at depth=500, float64 weights, clone to float32
2. Forward WITHOUT StabilizingNorm: measure relative error at each depth checkpoint (10, 50, 100, 200, 300, 400, 500) by running partial chains
3. Forward WITH StabilizingNorm(algebra.dim) applied every K layers: same depth checkpoints
4. Compute stable depth (max depth < 1e-3 relative error) for both baseline and mitigated
5. Report ratio: `mitigated_stable_depth / baseline_stable_depth` -- SC-4 criterion is ratio >= 2.0

Implementation: build the forward function to record intermediate errors at checkpoint depths:
```python
def forward_with_checkpoints(layers, x, checkpoint_depths, stabilizer=None, normalize_every=None):
    """Forward pass recording relative error at checkpoint depths."""
    errors = {}
    h = x
    for i, layer in enumerate(layers):
        h = layer(h)
        if stabilizer is not None and normalize_every is not None and (i + 1) % normalize_every == 0:
            h = stabilizer(h)
        depth = i + 1
        if depth in checkpoint_depths:
            errors[depth] = h.clone()
    return errors
```

**_algebra_rnn.py**: Algebra-specific recurrent cells following published designs (Parcollet QLSTM, Trabelsi complex RNN):

`RealLSTMCell(input_size, hidden_size, dtype=float32)`:
- Standard LSTM cell wrapping `nn.LSTMCell(input_size, hidden_size)`.
- Returns (h_t, c_t) tuple.

`ComplexGRUCell(input_size, hidden_size, dtype=float32)`:
- Complex-valued GRU following Trabelsi et al. pattern.
- Input: x shape [..., input_size, 2], h shape [..., hidden_size, 2].
- Gates (reset r, update z) computed via complex linear + sigmoid on magnitude.
- Candidate h_tilde computed via complex linear + complex tanh (apply tanh to each component).
- Parameters: 3 ComplexLinear layers for input, 3 for hidden (r, z, h gates each).
- Output: h_t shape [..., hidden_size, 2].

`QuaternionLSTMCell(input_size, hidden_size, dtype=float32)`:
- Quaternion LSTM following Parcollet et al. 2019 (ICLR).
- Input: x shape [..., input_size, 4], h shape [..., hidden_size, 4], c shape [..., hidden_size, 4].
- Gates: i, f, o computed via quaternion linear + split sigmoid (apply sigmoid per component, then take norm -- OR apply sigmoid to each component as in Parcollet).
- Follow Parcollet's approach: gates are real-valued scalars derived from quaternion computations. Specifically: i = sigmoid(real_part(W_i * x + U_i * h + b_i)), etc. The gating is done via the real component of the quaternion gate computation.
- Cell update: c_t = f * c_{t-1} + i * tanh(W_c * x + U_c * h), h_t = o * tanh(c_t). Multiplication by scalar gates broadcasts across quaternion components.
- Parameters: 4 QuaternionLinear layers for input gates (i, f, o, c), 4 for hidden state.
- Output: (h_t, c_t) tuple.

`OctonionLSTMCell(input_size, hidden_size, dtype=float32)`:
- Extends QuaternionLSTM pattern to 8 dimensions.
- Same gate structure: gates derived from real component of octonionic computation.
- Uses OctonionDenseLinear for the algebra-specific linear transforms.
- Parameters: 4 OctonionDenseLinear for input, 4 for hidden.
- Output: (h_t, c_t) tuple.

All recurrent cells follow the interface: `forward(x, state) -> (output, new_state)` where state is h or (h, c) depending on cell type.

**_network.py**: `AlgebraNetwork(config: NetworkConfig) -> nn.Module`

The key design: one class with three topology builders. The algebra type determines which specific layers are used, but the topology structure is identical.

```python
class AlgebraNetwork(nn.Module):
    def __init__(self, config: NetworkConfig):
        super().__init__()
        self.config = config
        # Auto-compute hidden size for parameter matching
        hidden = config.base_hidden * config.algebra.multiplier
        self.hidden = hidden
        self.algebra_dim = config.algebra.dim

        # Build topology-specific layers
        if config.topology == "mlp":
            self._build_mlp(config, hidden)
        elif config.topology in ("conv1d", "conv2d"):
            self._build_conv(config, hidden)
        elif config.topology == "recurrent":
            self._build_recurrent(config, hidden)
        else:
            raise ValueError(f"Unknown topology: {config.topology}")
```

**Input projection**: `nn.Linear(input_dim, hidden * algebra_dim)` followed by reshape to [..., hidden, algebra_dim]. This is a learned real-to-algebra embedding per CONTEXT.md decision.

**MLP topology** (`_build_mlp`):
- Input projection
- `depth` hidden blocks, each: AlgebraLinear(hidden, hidden) + AlgebraBN(hidden) + Activation
- Output projection

**Conv2D topology** (`_build_conv`):
- Initial conv block: AlgebraConv2d(input_channels, base_filters, 3, padding=1)
- `depth` conv blocks with residual connections where dimensions match: AlgebraConv2d + AlgebraBN + Activation. Double filters and halve spatial dims at stage boundaries (every depth//3 blocks).
- Global average pooling
- AlgebraLinear(final_filters, output_hidden) + output projection

**Recurrent topology** (`_build_recurrent`):
- Input projection
- Stack of `depth` algebra-specific RNN cells
- Take final hidden state -> output projection

**Output projection** (4 strategies per CONTEXT.md):
1. "real": Extract component 0: output[..., 0], then nn.Linear to output_dim
2. "flatten": Reshape [..., hidden, algebra_dim] -> [..., hidden * algebra_dim], then nn.Linear to output_dim
3. "norm": Compute algebra norm along last dim, then nn.Linear to output_dim
4. "learned": nn.Linear(hidden * algebra_dim, output_dim) -- same as flatten but named differently for phased training protocol

**Layer factory methods**: `_get_linear(in_f, out_f)`, `_get_conv2d(in_ch, out_ch, kernel)`, `_get_bn(features)`, `_get_activation()`, `_get_rnn_cell(in_size, hidden_size)` -- each dispatches on `self.config.algebra` to return the algebra-specific layer.

**param_report()** method: iterate `self.named_parameters()`, return list of dicts with layer name, shape, real param count, percentage.

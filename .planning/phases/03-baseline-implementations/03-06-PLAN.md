**_benchmarks.py**: Benchmark-specific data loading and configuration.

`build_cifar10_data(batch_size: int, data_dir: str = "./data") -> tuple[DataLoader, DataLoader, DataLoader]`:
- Load CIFAR-10 via `torchvision.datasets.CIFAR10`.
- Standard augmentation for train: RandomCrop(32, padding=4), RandomHorizontalFlip, Normalize((0.4914, 0.4822, 0.4465), (0.2023, 0.1994, 0.2010)).
- Val/test: Normalize only.
- Split: torchvision standard train/test split. Split 10% of train for validation.
- Return (train_loader, val_loader, test_loader).

`build_cifar100_data(batch_size: int, data_dir: str = "./data") -> tuple[DataLoader, DataLoader, DataLoader]`:
- Same as CIFAR-10 but CIFAR100 dataset. Different normalization stats.

`cifar_network_config(algebra: AlgebraType, dataset: str = "cifar10") -> NetworkConfig`:
- Build the Conv2D network config matching published architectures.
- For Gaudet and Maida reproduction: Deep residual architecture with 10+9+9 blocks per stage.
- For Trabelsi reproduction: Wide/Shallow or Deep/Narrow variant.
- Since both papers use ResNet-style architectures on CIFAR, use a common ResNet config:
  - topology="conv2d"
  - depth=28 (roughly matching the residual block count)
  - base_hidden=16 (standard CIFAR ResNet initial filters, will be auto-scaled per algebra)
  - activation="split_relu" (standard for CIFAR)
  - use_batchnorm=True
  - input_dim=3 (CIFAR channels)
  - output_dim=10 for CIFAR-10, 100 for CIFAR-100
  - output_projection="flatten"
- Key: the CIFAR input encoding for each algebra:
  - Real: standard 3-channel image, no encoding change
  - Complex: encode RGB as real+imaginary pair. Following Trabelsi: map 3 channels to 2-component complex (use learned 3->2 projection, or zero-pad to 4 channels then split into 2 complex)
  - Quaternion: Following Gaudet: encode as quaternion with real=grayscale (or zero), imag=RGB. q = 0 + R*i + G*j + B*k.
  - Octonion: Extend quaternion pattern: o = 0 + R*e1 + G*e2 + B*e3 + 0*e4 + 0*e5 + 0*e6 + 0*e7 (zero-pad remaining)
- Document the encoding choice in docstring and config.

`cifar_train_config(dataset: str = "cifar10") -> TrainConfig`:
- Match published hyperparameters first (per CONTEXT.md):
  - epochs=200 (standard for CIFAR benchmarks)
  - lr=0.01 (Gaudet/Trabelsi starting LR)
  - optimizer="sgd" (with momentum=0.9)
  - scheduler="cosine" (CosineAnnealingLR)
  - weight_decay=5e-4 (standard for CIFAR)
  - batch_size=128
  - warmup_epochs=5
  - early_stopping_patience=50 (generous for CIFAR)
  - use_amp=False (float32 for reproduction fidelity)

`reproduction_report(published: dict, ours: dict, output_path: str)`:
- Generate structured comparison document.
- Columns: Algebra, Published Error%, Our Error% (mean +/- std), Param Count (published), Param Count (ours), Pass/Fail (within 1 std).
- Save as JSON and markdown.

**Step 2: Run CIFAR-10 comparison** using `run_comparison`:

from octonion.baselines import run_comparison, ComparisonConfig, AlgebraType, TrainConfig
from octonion.baselines._benchmarks import build_cifar10_data, cifar_train_config

config = ComparisonConfig(
    task="cifar10_reproduction",
    algebras=[AlgebraType.REAL, AlgebraType.COMPLEX, AlgebraType.QUATERNION, AlgebraType.OCTONION],
    seeds=3,  # Start with 3 seeds for initial validation

    output_dir="experiments"

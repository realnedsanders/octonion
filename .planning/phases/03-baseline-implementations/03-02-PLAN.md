**_normalization.py**: Four batch normalization modules following Gaudet & Maida (H) and Trabelsi (C):

`RealBatchNorm(num_features, eps=1e-5, momentum=0.1)`: Wraps `nn.BatchNorm1d`. Standard real BN. 2 params/feature (gamma, beta).

`ComplexBatchNorm(num_features, eps=1e-5, momentum=0.1)`:
- Input shape: [batch, features, 2].
- Learnable: gamma_rr, gamma_ri, gamma_ii (3 per feature, symmetric 2x2 scale matrix) + beta (2 per feature, complex shift) = 5 params/feature.
- Running buffers: running_mean [features, 2], running_var_rr/ri/ii [features].
- Forward (train): Compute batch mean and 2x2 covariance V = [[Vrr, Vri], [Vri, Vii]]. Compute V^{-1/2} via analytic 2x2 inverse square root (use the formula for 2x2 positive definite matrices, no eigendecomposition needed). Apply: x_whitened = V^{-1/2} (x - mean), output = gamma @ x_whitened + beta.
- Forward (eval): Use running stats.

`QuaternionBatchNorm(num_features, eps=1e-5, momentum=0.1)`:
- Input shape: [batch, features, 4].
- Learnable: gamma [features, 4, 4] (initialized to identity, 10 unique entries for symmetric matrix) + beta [features, 4] = 14 params/feature.
- Forward: Compute 4x4 covariance, Cholesky decompose, whiten via triangular solve. Apply gamma and beta.
- Use `torch.linalg.cholesky` and `torch.linalg.solve_triangular` for numerical stability.

`OctonionBatchNorm(num_features, eps=1e-5, momentum=0.1)`:
- Input shape: [batch, features, 8].
- Learnable: gamma [features, 8, 8] (36 unique entries for symmetric) + beta [features, 8] = 44 params/feature.
- Forward: Same Cholesky whitening pattern as quaternion but for 8x8.
- If Cholesky fails (singular covariance), fall back to regularized covariance (add eps*I before decomposition). Log a warning if this happens.

For QuaternionBN and OctonionBN, store gamma as full square matrix but enforce symmetry: gamma_symmetric = (gamma + gamma.transpose(-1, -2)) / 2 in forward pass. This avoids constrained parameterization while ensuring correct whitening.

**_activation.py**: Two activation strategies per CONTEXT.md:

`SplitActivation(activation: str = "relu")`:
- Applies activation function independently to each algebra component.
- Supported: "relu", "gelu", "leaky_relu", "tanh", "silu".
- Forward: `F.{activation}(x)` -- trivial since PyTorch elementwise ops work on any shape.

`NormPreservingActivation(activation: str = "relu")`:
- Applies activation to the algebra norm, then rescales to preserve direction.
- Forward: compute norm = ||x|| along last dim, apply activation to norm (e.g., relu(norm)), compute scale = activated_norm / (norm + eps), output = x * scale.unsqueeze(-1).
- This preserves the algebraic direction while allowing gating of magnitude.

**_algebra_conv.py**: Algebra-specific convolutional layers following the same pattern as the linear layers:

`RealConv1d(in_channels, out_channels, kernel_size, stride=1, padding=0, bias=True, dtype=float32)`:
- Wraps `nn.Conv1d`. Standard real convolution. Init: Kaiming/He.

`RealConv2d(...)`: Wraps `nn.Conv2d`. Standard real convolution. Init: Kaiming/He.

`ComplexConv1d(in_channels, out_channels, kernel_size, stride=1, padding=0, bias=True, dtype=float32)`:
- Parameters: W_r, W_i as [out_channels, in_channels, kernel_size] conv weight tensors. Bias: bias_r, bias_i as [out_channels].
- Init: `complex_init` applied to reshaped weight (treat spatial dims as part of fan_in).
- Forward: input [B, in_ch, 2, L]. Extract real and imag channels. Apply complex conv: out_r = conv1d(x_r, W_r) - conv1d(x_i, W_i), out_i = conv1d(x_r, W_i) + conv1d(x_i, W_r). Stack components.
- Use `F.conv1d` with the raw weight tensors.

`ComplexConv2d(...)`: Same pattern for 2D convolution. Input [B, in_ch, 2, H, W].

`QuaternionConv1d(...)`:
- Parameters: W_r, W_i, W_j, W_k as conv weights. Bias: [out_channels, 4].
- Forward: Hamilton product convolution (16 terms, same sign pattern as QuaternionLinear).

`QuaternionConv2d(...)`: Same for 2D.

`OctonionConv1d(...)`:
- Parameters: W_0..W_7 as conv weights. Bias: [out_channels, 8].
- Forward: Use STRUCTURE_CONSTANTS for the full octonionic convolution product.

`OctonionConv2d(...)`: Same for 2D.

For all conv layers, the algebra dimension comes BEFORE the spatial dimensions in tensor layout: [B, channels, algebra_dim, *spatial]. This allows standard F.conv1d/F.conv2d to operate on each component pair.

The cholesky_ex change eliminates a torch.compile graph break AND provides more precise per-feature fallback instead of all-or-nothing exception handling. CLI flags make these features accessible without code edits.

**Step 2: Replace try/except cholesky with cholesky_ex in `_normalization.py`:**

In BOTH `QuaternionBatchNorm._whiten` AND `OctonionBatchNorm._whiten`, replace the try/except pattern with `torch.linalg.cholesky_ex`:

```python
def _whiten(self, x_centered: torch.Tensor, cov: torch.Tensor) -> torch.Tensor:
    # Disable autocast for numerically sensitive Cholesky/solve_triangular.
    # When AMP is enabled, inputs arrive as float16 but Cholesky needs float32.
    # When AMP is disabled, this context manager is a no-op.
    with torch.amp.autocast(device_type=x_centered.device.type, enabled=False):
        # Force float32 for all linalg operations
        cov_f32 = cov.float()
        x_f32 = x_centered.float()

        eye = torch.eye(
            self.dim, device=cov_f32.device, dtype=torch.float32
        ).unsqueeze(0)
        cov_reg = cov_f32 + self.eps * eye

        # Use cholesky_ex instead of try/except cholesky:
        # - No graph breaks (torch.compile compatible)
        # - Per-feature failure detection via info tensor
        # - info[i] == 0 means feature i succeeded, info[i] > 0 means failure
        L, info = torch.linalg.cholesky_ex(cov_reg)

        # Per-feature fallback: only re-regularize features that failed
        failed = info > 0  # [features] boolean mask
        if failed.any():
            num_failed = failed.sum().item()
            logger.warning(
                f"Cholesky failed for {num_failed}/{cov_reg.shape[0]} features, "
                "applying per-feature regularization."
            )
            # Apply stronger regularization only to failed features
            cov_fixed = cov_reg.clone()
            cov_fixed[failed] = cov_f32[failed] + 1e-1 * eye.squeeze(0)
            L_fixed, info_fixed = torch.linalg.cholesky_ex(cov_fixed)
            # Replace only failed features' L factors
            L = torch.where(failed.unsqueeze(-1).unsqueeze(-1), L_fixed, L)

            if (info_fixed[failed] > 0).any():
                # Last resort: if still failing after strong regularization,
                # use identity (effectively skip whitening for those features)
                still_failed = info_fixed > 0
                logger.warning(
                    f"Cholesky still failing for {still_failed.sum().item()} features "
                    "after strong regularization, using identity."
                )
                L[still_failed] = eye.squeeze(0) * (cov_f32[still_failed].diagonal(dim1=-2, dim2=-1).abs().mean(dim=-1, keepdim=True).unsqueeze(-1).clamp(min=1.0).sqrt())

        # Condition number monitoring (unchanged)
        if self.training:
            with torch.no_grad():
                diag = L.diagonal(dim1=-2, dim2=-1)
                diag_abs = diag.abs().clamp(min=1e-12)
                per_feature_cond = diag_abs.max(dim=-1).values / diag_abs.min(dim=-1).values
                self.last_cond.fill_(per_feature_cond.max().item())

        identity = eye.expand_as(L)
        L_inv = torch.linalg.solve_triangular(L, identity, upper=False)
        x_col = x_f32.unsqueeze(-1)
        w = (L_inv.unsqueeze(0) @ x_col).squeeze(-1)

    # Return in original dtype (autocast handles downstream casting)
    return w.to(x_centered.dtype)
```

Key details:
- `torch.linalg.cholesky_ex` returns `(L, info)` where `info` is an int tensor. `info[i] == 0` means success, `info[i] > 0` means the i-th feature's matrix failed at the info[i]-th step.
- The `failed` mask identifies exactly which features need re-regularization, avoiding the all-or-nothing try/except approach.
- Use `device_type=x_centered.device.type` NOT hardcoded `"cuda"` -- this makes the code work on both CPU and CUDA.
- Cast to `.float()` explicitly at the top of the block.
- Return with `.to(x_centered.dtype)` to restore the original dtype for downstream operations.
- The condition number monitoring is preserved.
- NO try/except blocks around cholesky calls -- this is critical for torch.compile compatibility.

**Step 3: Also protect the covariance computation in `forward()`:**

In both QuaternionBatchNorm.forward and OctonionBatchNorm.forward, the covariance is computed via `einsum("bfi,bfj->fij", x_centered, x_centered) / (N - 1)`. This batch accumulation may lose precision in float16 for large batches. Wrap the covariance computation in the same autocast disable block, or explicitly cast the inputs to float32 for the einsum.

The simplest approach: in the `forward()` method, right before computing covariance, add:
```python
# Force float32 for covariance computation (accumulation over batch)
x_centered_f32 = x_centered.float()
cov = torch.einsum("bfi,bfj->fij", x_centered_f32, x_centered_f32) / (N - 1)
```
This ensures the covariance matrix is always float32 before being passed to `_whiten`.

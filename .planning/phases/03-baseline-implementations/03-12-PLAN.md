**Step 2: Add AMP protection to `_whiten` in `_normalization.py`:**

In BOTH `QuaternionBatchNorm._whiten` AND `OctonionBatchNorm._whiten` (they share the same pattern), wrap the numerically sensitive operations in an autocast disable block:

```python
def _whiten(self, x_centered: torch.Tensor, cov: torch.Tensor) -> torch.Tensor:
    # Disable autocast for numerically sensitive Cholesky/solve_triangular.
    # When AMP is enabled, inputs arrive as float16 but Cholesky needs float32.
    # When AMP is disabled, this context manager is a no-op.
    with torch.amp.autocast(device_type=x_centered.device.type, enabled=False):
        # Force float32 for all linalg operations
        cov_f32 = cov.float()
        x_f32 = x_centered.float()

        eye = torch.eye(
            self.dim, device=cov_f32.device, dtype=torch.float32
        ).unsqueeze(0)
        cov_reg = cov_f32 + self.eps * eye

        try:
            L = torch.linalg.cholesky(cov_reg)
        except torch.linalg.LinAlgError:
            logger.warning(
                "Cholesky decomposition failed, increasing regularization to 1e-1."
            )
            cov_reg = cov_f32 + 1e-1 * eye
            L = torch.linalg.cholesky(cov_reg)

        # Condition number monitoring (unchanged)
        if self.training:
            with torch.no_grad():
                diag = L.diagonal(dim1=-2, dim2=-1)
                diag_abs = diag.abs().clamp(min=1e-12)
                per_feature_cond = diag_abs.max(dim=-1).values / diag_abs.min(dim=-1).values
                self.last_cond.fill_(per_feature_cond.max().item())

        identity = eye.expand_as(L)
        L_inv = torch.linalg.solve_triangular(L, identity, upper=False)
        x_col = x_f32.unsqueeze(-1)
        w = (L_inv.unsqueeze(0) @ x_col).squeeze(-1)

    # Return in original dtype (autocast handles downstream casting)
    return w.to(x_centered.dtype)
```

Key details:
- Use `device_type=x_centered.device.type` NOT hardcoded `"cuda"` -- this makes the code work on both CPU and CUDA.
- Cast to `.float()` explicitly at the top of the block.
- Return with `.to(x_centered.dtype)` to restore the original dtype for downstream operations.
- The existing `try/except` Cholesky fallback is preserved.
- The existing condition number monitoring is preserved.

**Step 3: Also protect the covariance computation in `forward()`:**

In both QuaternionBatchNorm.forward and OctonionBatchNorm.forward, the covariance is computed via `einsum("bfi,bfj->fij", x_centered, x_centered) / (N - 1)`. This batch accumulation may lose precision in float16 for large batches. Wrap the covariance computation in the same autocast disable block, or explicitly cast the inputs to float32 for the einsum.

The simplest approach: in the `forward()` method, right before computing covariance, add:
```python
# Force float32 for covariance computation (accumulation over batch)
x_centered_f32 = x_centered.float()
cov = torch.einsum("bfi,bfj->fij", x_centered_f32, x_centered_f32) / (N - 1)
```
This ensures the covariance matrix is always float32 before being passed to `_whiten`.

# Optional torch.compile for potential kernel fusion speedup
if getattr(config, 'use_compile', False) and str(device).startswith("cuda"):
    try:
        model = torch.compile(model, backend="inductor", mode="default")

Using `getattr` with a default ensures backward compatibility if someone loads an old TrainConfig without `use_compile`.

The `try/except` is important because torch.compile may fail on ROCm with certain model patterns (graph breaks from ParameterList, etc.). Falling back to eager mode is safe.

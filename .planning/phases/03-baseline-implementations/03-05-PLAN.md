  <action>
**_comparison.py**:

`@dataclass ComparisonReport`:
- task: str
- algebras: list[str]
- seeds: int
- per_run: list[dict] -- each has algebra, seed, metrics (train_losses, val_losses, val_accuracies, best_val_acc, total_time)
- pairwise: dict[str, dict] -- keyed by "R_vs_C" etc., values are paired_comparison results
- corrected_pairwise: list[dict] -- Holm-Bonferroni corrected results
- param_counts: dict[str, int] -- per-algebra total param count
- config_hash: str -- hash of full config for reproducibility tracking
- timestamp: str

`run_comparison(task_name: str, build_data_fn: Callable, config: ComparisonConfig, device: str = "cuda") -> ComparisonReport`:

**Parameters**:
- `task_name`: experiment name (e.g., "cifar10_conv2d")
- `build_data_fn`: callable returning (train_loader, val_loader, test_loader, input_dim, output_dim, input_channels). This keeps data loading task-specific and out of the runner.
- `config`: ComparisonConfig with algebras, seeds, train_config, output_dir.
- `device`: "cuda" or "cpu".

**Execution flow**:
1. Create output directory: `{config.output_dir}/{task_name}/`.
2. Build reference model (octonion) to get target param count.
3. Use `find_matched_width` to compute hidden widths for each algebra.
4. Log param counts and verify all within 1% tolerance. Raise ValueError if any fail.
5. Save model summaries via `torchinfo.summary()` to `{task_dir}/{algebra}/model_summary.txt`.
6. For each algebra in config.algebras:
   For each seed in range(config.seeds):
     a. Set seed via `seed_everything(seed)`.
     b. Build AlgebraNetwork with computed hidden width.
     c. Create experiment dir: `{task_dir}/{algebra.name}/{seed}/`.
     d. Save config.json (serialized TrainConfig + NetworkConfig).
     e. Build data loaders via `build_data_fn(config.train_config.batch_size)`.
     f. Call `train_model(model, train_loader, val_loader, config.train_config, experiment_dir, device)`.
     g. Save metrics.json with full results.
     h. Generate per-run convergence plot via `plot_convergence`.
     i. Collect final metrics.
7. After all runs complete:
   a. Compute pairwise statistical comparisons for all algebra pairs using best_val_acc.
   b. Apply Holm-Bonferroni correction.
   c. Generate comparison bar plot.
   d. Generate param table plot.
   e. Update manifest.json.
   f. Build and return ComparisonReport.

**Manifest management**:
`_update_manifest(output_dir, task_name, report)`:
- Load existing manifest.json or create new.
- Append entry with config_hash, timestamp, task_name, final metrics summary, status ("complete").
- Save back.

**Config hashing**: Hash the JSON serialization of ComparisonConfig (deterministic) for reproducibility tracking.

**Sequential execution** per CONTEXT.md: one model trains at a time. No multiprocessing. Simple for loop.

**tests/test_baselines_comparison.py**: Use a tiny MLP with synthetic data (2D input, 2-class classification) for fast tests:
- `test_run_comparison_creates_directories`: Run with 2 algebras x 1 seed, verify directory structure
- `test_run_comparison_returns_report`: Verify ComparisonReport has all expected fields
- `test_manifest_created`: Verify manifest.json exists and contains entry
- `test_pairwise_stats_computed`: Verify pairwise comparison dict has entries for each pair
- `test_param_counts_within_tolerance`: Verify all algebras within 1% of each other
- `test_plots_generated`: Verify PNG files exist in output directory
- `test_config_json_saved`: Verify config.json in each experiment dir
- `test_metrics_json_saved`: Verify metrics.json in each experiment dir

NOTE: Tests should use very small models (base_hidden=4, depth=1) and few epochs (2-3) to run in seconds, not minutes.

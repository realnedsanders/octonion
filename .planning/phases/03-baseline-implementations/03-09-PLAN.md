<task type="auto">
  <name>Task 1: Create CIFAR reproduction script and execute training</name>
  <files>scripts/run_cifar_reproduction.py</files>
  <action>
Create `scripts/run_cifar_reproduction.py` -- a standalone script that orchestrates the full CIFAR-10 benchmark reproduction using the existing infrastructure.

The script should:

1. **Import the existing infrastructure** -- `run_comparison`, `build_cifar10_data`, `cifar_train_config`, `reproduction_report`, `PUBLISHED_RESULTS`, `AlgebraType`, `ComparisonConfig`.

2. **Configure the CIFAR-10 experiment**:
   - `task_name = "cifar10_reproduction"`
   - `algebras = [AlgebraType.REAL, AlgebraType.COMPLEX, AlgebraType.QUATERNION, AlgebraType.OCTONION]`
   - `seeds = 3` (3 seeds for initial validation -- sufficient for mean+std)
   - `train_config = cifar_train_config("cifar10")` (200 epochs, SGD momentum=0.9, cosine LR)
   - `network_config_overrides = {"topology": "conv2d", "depth": 28, "ref_hidden": 16}` (ref_hidden=16 matches cifar_network_config base_hidden=16 for standard CIFAR ResNet filter count)
   - `output_dir = "experiments"`

3. **Run the comparison**: Call `run_comparison(task_name, build_cifar10_data, config, device="cuda", network_config_overrides=overrides)`. This handles parameter matching, training all 12 runs (4 algebras x 3 seeds), statistical testing, plot generation, and manifest update.

4. **Generate the reproduction report**: After `run_comparison` returns, extract per-algebra results from `report.per_run`:
   - For each algebra, compute `mean_error = mean((1 - best_val_acc) * 100)` and `std_error` across seeds
   - Build the `ours` dict with keys: `error_pct`, `std_pct`, `param_count`, `seeds`, `per_seed_errors`
   - Call `reproduction_report(PUBLISHED_RESULTS["cifar10"], ours, "experiments/cifar10_reproduction/reproduction_report")`

5. **Print a summary** to stdout with the verdicts table and overall pass/fail.

6. **Add argument parsing** with `argparse`:
   - `--seeds N` (default 3)
   - `--device` (default "cuda")
   - `--data-dir` (default "./data")
   - `--output-dir` (default "experiments")
   - `--ref-hidden` (default 16)
   - `--num-workers` (default 4, for data loader)

7. **Add logging setup**: Configure Python logging at INFO level so `run_comparison` log messages are visible.

8. **Handle graceful SIGINT**: The trainer already has SIGINT handling for checkpoint saving. The script should print a message about checkpoint resumption on interrupt.

IMPORTANT: The script runs INSIDE the container. The command is:
```bash
docker compose run --rm dev uv run python scripts/run_cifar_reproduction.py --seeds 3
```

After creating the script, execute the training by running the above command. This will take several hours (200 epochs x 4 algebras x 3 seeds). The training will produce:
- `experiments/cifar10_reproduction/{REAL,COMPLEX,QUATERNION,OCTONION}/{0,1,2}/` directories with checkpoints, metrics, convergence plots
- `experiments/cifar10_reproduction/comparison_accuracy.png`
- `experiments/cifar10_reproduction/param_table.png`
- `experiments/cifar10_reproduction/reproduction_report.{json,md}`
- `experiments/manifest.json`

If any algebra fails to reproduce published results (outside 1 std):
- Check the reproduction_report.md for the specific failure
- The most likely causes: wrong ref_hidden (try ref_hidden=4 matching existing test scaffolds), learning rate too high/low, or insufficient epochs
- If needed, rerun with adjusted `--ref-hidden` value
- Document both results per CONTEXT.md: "match published hyperparams first, then tune if reproduction fails"

NOTE: If GPU is unavailable or training fails to start, the script should fall back to `--device cpu` with a warning (CPU training will be very slow but should work for validation).

<objective>
Apply Tier 2 (moderate-risk) performance optimizations: fuse the OctonionDenseLinear forward pass from a 64-iteration Python loop to a single einsum+F.linear call, add eval-mode fused weight caching in conv layers, and register structure constants as module buffers. These optimizations produce mathematically identical results but change the computational approach, requiring equivalence tests.

Purpose: The Python-loop OctonionDenseLinear forward and per-forward fused weight recomputation in conv layers are the two largest remaining bottlenecks after Tier 1. The fused einsum pattern is already proven in OctonionConv2d -- applying it to the linear layer is a direct analogy. Eval-mode caching eliminates redundant computation during validation epochs.

Output: Optimized forward passes with equivalence tests proving identical outputs.
</objective>

def forward(self, x: torch.Tensor) -> torch.Tensor:
    batch_shape = x.shape[:-2]
    linear_cache: dict[tuple[int, int], torch.Tensor] = {}
    out_components = [
        torch.zeros(*batch_shape, self.out_features, dtype=x.dtype, device=x.device)
        for _ in range(8)
    ]
    for i, j, k, coeff in self._nonzero_entries:
        key = (i, j)
        if key not in linear_cache:
            linear_cache[key] = F.linear(x[..., j], self.weights[i])
        out_components[k] = out_components[k] + coeff * linear_cache[key]
    result = torch.stack(out_components, dim=-1)
    if self.bias is not None:
        result = result + self.bias
    return result

**Step 2: Fuse OctonionDenseLinear forward in `_algebra_linear.py`:**

Replace the Python-loop forward (lines 279-313) with the fused einsum pattern from OctonionConv2d:

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    batch_shape = x.shape[:-2]

    # Stack weights: [8, out_f, in_f]
    W_stack = torch.stack(list(self.weights))

    # Fuse with structure constants:
    # fused[k, j] = sum_i C[i,j,k] * W_i  ->  [8, 8, out_f, in_f]
    fused = torch.einsum("ijk, iof -> kjof", self._C, W_stack)

    # Reshape to [8*out_f, 8*in_f] for single F.linear
    fused_flat = fused.permute(0, 2, 1, 3).reshape(
        8 * self.out_features, 8 * self.in_features
    )

    # Flatten input: [..., in_f, 8] -> [..., 8*in_f]
    x_flat = x.reshape(*batch_shape, 8 * self.in_features)

    # Single F.linear call
    out_flat = F.linear(x_flat, fused_flat)

    # Reshape: [..., 8*out_f] -> [..., out_f, 8]
    result = out_flat.reshape(*batch_shape, self.out_features, 8)

    if self.bias is not None:
        result = result + self.bias

    return result
```

**Add eval-mode caching to OctonionConv2d:**

During evaluation (model.eval()), weights don't change between batches, so the fused weight matrix can be computed once and reused for all validation batches. This eliminates `torch.stack` + `torch.einsum` + reshape per forward call during validation.

In `OctonionConv2d`:

1. Add `self._fused_cache = None` in `__init__`.

2. Override `train()` to invalidate the cache:
```python
def train(self, mode=True):
    super().train(mode)
    if mode:
        self._fused_cache = None
    return self
```

3. In `forward()`, wrap the fused weight computation in a cache check:
```python
if not self.training and self._fused_cache is not None:
    fused = self._fused_cache
else:
    W_stack = torch.stack(list(self.weights))
    fused_blocks = torch.einsum("ijk, iochw -> kjochw", self._C, W_stack)
    fused = fused_blocks.permute(0, 2, 1, 3, 4, 5).reshape(
        8 * out_ch, 8 * in_ch, *self._kernel_size
    )
    if not self.training:
        self._fused_cache = fused
```

**Apply the same pattern to QuaternionConv2d** (which uses Hamilton product cat instead of einsum, but the same caching principle applies -- cache the constructed weight matrix during eval).

**Apply the same pattern to OctonionConv1d and QuaternionConv1d** if they exist.

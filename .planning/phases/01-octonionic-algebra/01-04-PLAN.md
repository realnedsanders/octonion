**Fix 2: Near-zero suppression in __str__**

In `Octonion.__str__`, replace the `val != 0.0` check on line 234 with a dtype-aware threshold. For imaginary components, suppress values below a reasonable display threshold:

```python
def __str__(self) -> str:
    if self._data.dim() > 1:
        return f"Octonion(shape={list(self._data.shape[:-1])}, dtype={self._data.dtype})"

    # Dtype-aware display threshold: suppress near-zero noise
    # float32 eps ~1.2e-7, float64 eps ~2.2e-16
    atol = 1e-7 if self._data.dtype == torch.float32 else 1e-14

    parts = []
    for i in range(8):
        val = self._data[i].item()
        if i == 0:
            parts.append(f"{val}")
        elif abs(val) > atol:
            sign = " + " if val > 0 else " - "
            parts.append(f"{sign}{abs(val)}*{_BASIS_LABELS[i]}")
    return "".join(parts) if parts else "0.0"
```

This ensures float32 residuals like 1.5e-08 are suppressed while genuine values are preserved.

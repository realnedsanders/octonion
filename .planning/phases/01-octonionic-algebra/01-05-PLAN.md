def octonion_mul(a: torch.Tensor, b: torch.Tensor) -> torch.Tensor:
    # Promote a and b to common dtype (handles float32/float64 mismatch)
    common_dtype = torch.promote_types(a.dtype, b.dtype)
    a = a.to(dtype=common_dtype)
    b = b.to(dtype=common_dtype)
    C = STRUCTURE_CONSTANTS.to(device=a.device, dtype=common_dtype)

**Fix: Add auto-coercion guard to octonion_exp and octonion_log**

In `src/octonion/_operations.py`, add a type guard at the top of both `octonion_exp` and `octonion_log` that wraps raw tensors in Octonion:

For `octonion_exp`:
```python
def octonion_exp(o: Octonion | torch.Tensor) -> Octonion:
    """..."""
    if isinstance(o, torch.Tensor) and not isinstance(o, Octonion):
        o = Octonion(o)
    a = o.real
    v = o.imag
    ...  # rest unchanged
```

For `octonion_log`:
```python
def octonion_log(o: Octonion | torch.Tensor) -> Octonion:
    """..."""
    if isinstance(o, torch.Tensor) and not isinstance(o, Octonion):
        o = Octonion(o)
    a = o.real
    v = o.imag
    ...  # rest unchanged
```

The `not isinstance(o, Octonion)` guard is needed because Octonion is NOT a subclass of torch.Tensor, so this is just checking "is it a plain tensor?" vs "is it already an Octonion?". Since Octonion does not inherit from Tensor, the check `isinstance(o, torch.Tensor)` will be False for Octonion instances. So the simpler form also works:

```python
if not isinstance(o, Octonion):
    o = Octonion(o)
```

Use whichever is clearer. Update the type annotation of the `o` parameter to `Octonion | torch.Tensor` in both functions.

Also update the `from __future__ import annotations` import is already present (it is), so the union type hint works.

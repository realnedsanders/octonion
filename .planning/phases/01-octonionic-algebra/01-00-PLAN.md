  <name>Task 1: Containerfile and docker-compose.yml for ROCm PyTorch development</name>
  <files>Containerfile, docker-compose.yml, scripts/container-shell.sh</files>
  <action>
1. Create `Containerfile` (Podman-compatible Dockerfile) based on the official ROCm PyTorch image:
   - Base image: `rocm/pytorch:rocm7.2_ubuntu24.04_py3.12_pytorch_release_2.9.1`
     (Ubuntu 24.04, Python 3.12, PyTorch 2.9.1, ROCm 7.2 -- matches project requirements of Python 3.12 and PyTorch >=2.7)
   - Install `uv` via the official installer: `curl -LsSf https://astral.sh/uv/install.sh | sh`
   - Set environment variables:
     - `UV_LINK_MODE=copy` (required for uv inside containers with mounted volumes)
     - `PATH` includes `$HOME/.local/bin` (where uv installs)
     - `HSA_OVERRIDE_GFX_VERSION` left unset (user sets per their GPU if needed)
   - Set working directory to `/workspace`
   - Do NOT copy project source into the image -- it will be mounted via docker-compose
   - Do NOT install project dependencies in the image -- uv sync will handle that at runtime from the mounted source
   - Add a `CMD` that runs `bash` for interactive use
   - Add labels: maintainer, description, rocm version, pytorch version

2. Create `docker-compose.yml`:
   - Service name: `dev`
   - Build context: `.` with dockerfile: `Containerfile`
   - Volume mount: `.:/workspace` (project root into /workspace)
   - Volume mount: named volume `uv-cache:/root/.cache/uv` (persist uv cache across container restarts)
   - GPU passthrough for ROCm:
     - `devices: ["/dev/kfd", "/dev/dri"]` (AMD GPU kernel fusion driver and DRI)
     - `group_add: ["video", "render"]` (required for GPU access)
     - `security_opt: ["seccomp=unconfined"]` (required by ROCm)
   - `ipc: host` (required for PyTorch multiprocessing/shared memory)
   - `stdin_open: true` and `tty: true` for interactive shell
   - Environment variables:
     - `HSA_OVERRIDE_GFX_VERSION=${HSA_OVERRIDE_GFX_VERSION:-}` (passthrough from host, empty default)
   - Working directory: `/workspace`
   - Command: `bash`

3. Create `scripts/container-shell.sh`:
   - Shebang: `#!/usr/bin/env bash`
   - Set `-euo pipefail`
   - `cd` to project root (dirname of script, then up one level)
   - Run `docker compose run --rm dev bash` (or `podman compose` if docker not found)
   - Add a comment explaining: "Run this to get an interactive shell inside the dev container with GPU access"
   - Make the script executable: `chmod +x scripts/container-shell.sh`

IMPORTANT: Use `Containerfile` (not `Dockerfile`) as the filename -- this is the Podman-native naming and works with both Docker and Podman.
IMPORTANT: The base image already has PyTorch and ROCm pre-installed. Do NOT reinstall them via pip/uv.
IMPORTANT: The base image's Python/PyTorch should be used directly. The project's pyproject.toml will specify `torch>=2.7` which is already satisfied by the base image's PyTorch 2.9.1.
  </action>

**`src/octonion/calculus/_composition.py`**:

1. Binary tree types using frozen dataclasses (per CONTEXT.md: tree data structures, not string parsing):
```python
@dataclass(frozen=True)
class Leaf:
    index: int  # operand index (0-based)

@dataclass(frozen=True)
class Node:
    op: str  # operation name: "mul", "exp", "log", "conjugate", "inverse"
    left: Leaf | Node
    right: Leaf | Node | None  # None for unary ops (exp, log, conjugate, inverse)
```

2. `all_parenthesizations(n: int, op: str = "mul") -> list[Node]`: Generate all C_{n-1} binary tree structures for n operands in a multiplication chain. Standard recursive Catalan generation:
   - n=1: return [Leaf(0)]
   - n>1: for split in range(1, n), combine all_parenthesizations(split) with all_parenthesizations(n-split), reindexing right subtree by +split

3. `evaluate_tree(tree: Leaf | Node, operands: list[torch.Tensor]) -> torch.Tensor`: Recursively evaluate the binary tree. For each Node, evaluate left and right subtrees, then apply the operation using the autograd Function (OctonionMulFunction.apply for "mul", OctonionExpFunction.apply for unary ops, etc.). This ensures the computation graph tracks parenthesization for correct backward passes.

4. `CompositionBuilder` class:
```python
class CompositionBuilder:
    """Build and evaluate parenthesized octonionic compositions."""
    def __init__(self, tree: Leaf | Node):
        self.tree = tree

    @classmethod
    def from_parenthesization(cls, n: int, index: int) -> "CompositionBuilder":
        """Create from the index-th parenthesization of n operands."""
        trees = all_parenthesizations(n)
        return cls(trees[index])

    def evaluate(self, operands: list[torch.Tensor]) -> torch.Tensor: ...
    def __call__(self, operands: list[torch.Tensor]) -> torch.Tensor: ...
    def inspect(self) -> str:
        """Return ASCII tree representation."""
```

5. Support mixed operations: `build_mixed_tree(ops: list[str], structure: Leaf | Node) -> Node` that replaces "mul" op labels with the specified operation sequence. This enables testing compositions like mul -> exp -> log -> conjugate -> inverse.

**`src/octonion/calculus/_chain_rule.py`**:

1. `compose_jacobians(tree: Leaf | Node, operands: list[torch.Tensor]) -> list[torch.Tensor]`:
   Compute the Jacobian of the full composition w.r.t. each operand by traversing the tree bottom-up. At each internal node, the chain rule multiplies the node's operation Jacobian with the subtree Jacobians:
   - For Node(mul, left, right): J_total_wrt_x = J_mul_wrt_a @ J_left_wrt_x + J_mul_wrt_b @ J_right_wrt_x
   - Matrix multiplication of 8x8 Jacobians IS associative, so the chain rule application order doesn't matter -- only the tree structure (which determines which operations' Jacobians participate) matters.

2. `naive_chain_rule_jacobian(operands: list[torch.Tensor]) -> list[torch.Tensor]`:
   Compute gradients assuming left-to-right association: ((o1 * o2) * o3) * o4 * ... regardless of the actual tree structure. This is the "naive" baseline that ignores parenthesization. For comparison with the correct parenthesization-aware result.

**`src/octonion/calculus/_inspector.py`**:

1. `tree_to_string(tree: Leaf | Node) -> str`: Convert tree to mathematical notation, e.g., `"((x0 * x1) * x2)"` or `"(x0 * (x1 * x2))"`.

2. `inspect_tree(tree: Leaf | Node) -> str`: ASCII art tree display:

**OctonionMulFunction**: For f(a, b) = a * b:
```python
class OctonionMulFunction(torch.autograd.Function):
    @staticmethod
    def forward(ctx, a, b):
        ctx.save_for_backward(a, b)
        return octonion_mul(a, b)

    @staticmethod
    def backward(ctx, grad_output):
        a, b = ctx.saved_tensors
        C = STRUCTURE_CONSTANTS.to(device=a.device, dtype=a.dtype)
        # grad_a[i] = sum_k grad_output[k] * J_a[k,i] = sum_k grad_output[k] * sum_j C[i,j,k] * b[j]
        grad_a = torch.einsum("...k, ijk, ...j -> ...i", grad_output, C, b)
        # grad_b[j] = sum_k grad_output[k] * J_b[k,j] = sum_k grad_output[k] * sum_i C[i,j,k] * a[i]
        grad_b = torch.einsum("...k, ijk, ...i -> ...j", grad_output, C, a)
        return grad_a, grad_b
```
The backward uses einsum operations which are themselves differentiable by PyTorch, so create_graph=True automatically works for double backward (Hessian computation).

**OctonionExpFunction**: Save input o in forward. In backward, recompute the Jacobian (using the formula from _jacobians.py, but inline for differentiability -- do NOT call jacobian_exp which may detach). The backward computes `grad_input = einsum("...k, ...ki -> ...i", grad_output, J)` where J is the 8x8 exp Jacobian computed from saved input o. All intermediate computations (exp, sin, cos, etc.) must be differentiable for create_graph=True.

**OctonionLogFunction**: Same pattern as exp. Save input, recompute Jacobian in backward using differentiable operations.

**OctonionConjugateFunction**: Trivial: grad_input = grad_output * [1, -1, -1, -1, -1, -1, -1, -1]. No saved tensors needed.

**OctonionInverseFunction**: Save input o. In backward, compute J from the inverse Jacobian formula using differentiable operations.

**OctonionInnerProductFunction**: For f(a, b) = sum(a*b). grad_a = grad_output * b (broadcast grad_output [...] to [..., 8]). grad_b = grad_output * a.

**OctonionCrossProductFunction**: Save inputs a, b. Uses mul Jacobian applied to pure imaginary projections. More complex but follows from the chain rule: cross(a,b) = ImProj(mul(ImProj(a), ImProj(b))). The Jacobian is the composition of mul Jacobian with ImProj.

**Critical patterns (per RESEARCH.md anti-patterns):**
- Only save inputs in ctx.save_for_backward() -- never intermediate computations from forward (which runs in no-grad mode and would break create_graph=True)
- All backward operations must use differentiable PyTorch ops (einsum, exp, cos, sin, etc.) for double backward
- Use `STRUCTURE_CONSTANTS.to(device=a.device, dtype=a.dtype)` in backward (not cached from forward)

Create `src/octonion/calculus/_gradcheck.py` with:

**`octonion_gradcheck(fn, inputs, eps=1e-7, atol=1e-5, rtol=1e-3) -> dict`**:
Unlike torch.autograd.gradcheck, this:
1. Reports per-component octonionic errors (e0, e1, ..., e7) -- not just a single pass/fail
2. Validates both Wirtinger derivatives using `wirtinger_from_jacobian` from `_ghr.py`:
   - Compute numeric 8x8 Jacobian via central differences
   - Compute autograd 8x8 Jacobian by backpropagating unit vectors
   - Convert both to Wirtinger pairs and compare
3. Returns structured dict with: `passed` (bool), `max_abs_error`, `max_rel_error`, `per_component_max` (list of 8 floats), `wirtinger_passed` (bool), `wirtinger_error` (float)

Implementation approach:
- For each input with requires_grad=True, compute column-by-column Jacobian: set grad_output = e_k (unit vector in direction k), run backward, collect grad. This gives J_autograd[k, :] = backward(e_k).
- Compute J_numeric via `numeric_jacobian` or `numeric_jacobian_2arg` from `_numeric.py`
- Compare column-wise for per-component reporting
- Convert both J matrices to Wirtinger pairs via `wirtinger_from_jacobian` and compare

**`octonion_gradgradcheck(fn, inputs, eps=1e-7, atol=1e-5, rtol=1e-3) -> dict`**:
Verifies second-order derivatives by calling the function with create_graph=True in the first backward, then computing gradients of the gradient. Essentially wraps torch.autograd.gradgradcheck with octonionic reporting.

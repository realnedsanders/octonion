**`src/octonion/calculus/_analyticity.py`**:

Octonionic CR-like (Cauchy-Riemann) conditions. In complex analysis, a function f(z) is analytic iff its Jacobian (2x2 real matrix) has a specific structure: [[a, -b], [b, a]] (i.e., satisfies the CR equations). The octonionic analogue:

1. `cauchy_riemann_octonion(J: torch.Tensor) -> torch.Tensor`:
   Given an 8x8 real Jacobian J, compute the CR residual. An octonionic function f is "octonionic analytic" (or "regular") if its Jacobian can be represented as left multiplication by some octonion -- i.e., J = L_c for some c. The CR condition is: J must equal `left_mul_matrix` of some octonion c.

   Compute: For J to be L_c, we need J[k, j] = sum_i c[i] * C[i, j, k]. Extract the putative c from the first column of J: c = J[:, 0] (since L_c[:, 0] = c * e_0 = c). Then check if J == L_c for this c. The residual is ||J - L_c||_F.

   Return residual tensor (scalar or batched).

2. `analyticity_residual(fn: Callable, x: torch.Tensor, eps: float = 1e-7) -> torch.Tensor`:
   Compute the CR residual for function fn at point x by computing the numeric Jacobian and calling cauchy_riemann_octonion.

3. `is_octonionic_analytic(fn: Callable, x: torch.Tensor, tol: float = 1e-6) -> bool`:
   Returns True if analyticity_residual < tol.

Module docstring: Explain that very few octonionic functions are "analytic" in this sense -- the octonionic Cauchy-Riemann conditions are extremely restrictive. Left multiplication by a fixed octonion is analytic. Right multiplication is NOT (its Jacobian is R_b, not L_c). Conjugation is NOT. exp/log are NOT. This is a known feature of octonionic analysis and contrasts with complex analysis where many common functions are analytic.

**`src/octonion/calculus/_lr_scaling.py`**:

Learning rate scaling heuristic based on gradient magnitude characteristics.

1. `gradient_magnitude_stats(layer: nn.Module, n_samples: int = 1000, input_range: tuple[float, float] = (-1.0, 1.0), dtype: torch.dtype = torch.float64) -> dict`:
   Generate n_samples random inputs, compute gradients through the layer, return statistics:
   ```python
   {
       "grad_norm_mean": float,    # Mean of ||grad||
       "grad_norm_std": float,     # Std of ||grad||
       "grad_norm_max": float,     # Max ||grad||
       "grad_norm_min": float,     # Min ||grad||
       "grad_per_component": list[float],  # Mean |grad_i| for each of 8 components
       "ratio_to_real": float,     # ||O_grad|| / ||R8_grad|| comparing octonionic gradient to simple R^8 gradient
   }
   ```

2. `lr_scaling_heuristic(stats: dict) -> float`:
   Recommend a learning rate scaling factor. The heuristic: if octonionic gradient norms are K times larger than real-valued gradient norms on average, scale the learning rate by 1/K to maintain comparable update magnitudes.

3. `suggest_lr(base_lr: float, layer: nn.Module, n_samples: int = 500) -> float`:
   Convenience function: compute stats, apply heuristic, return adjusted learning rate.

import torch

GPU_AVAILABLE = torch.cuda.is_available()
gpu = pytest.mark.skipif(not GPU_AVAILABLE, reason="No GPU available")

@gpu
class TestGPUCPUParity:
    """Verify backward passes produce identical results on GPU and CPU."""

    def _compare_gradients(self, fn_cpu, fn_gpu, inputs_cpu, inputs_gpu, tol=1e-12):
        """Run forward+backward on CPU and GPU, compare gradients."""
        # CPU
        out_cpu = fn_cpu(*inputs_cpu)
        loss_cpu = out_cpu.sum()
        loss_cpu.backward()
        grads_cpu = [inp.grad.clone() for inp in inputs_cpu if inp.grad is not None]

        # GPU
        out_gpu = fn_gpu(*inputs_gpu)
        loss_gpu = out_gpu.sum()
        loss_gpu.backward()
        grads_gpu = [inp.grad.cpu().clone() for inp in inputs_gpu if inp.grad is not None]

        # Compare
        for i, (gc, gg) in enumerate(zip(grads_cpu, grads_gpu)):
            diff = torch.abs(gc - gg)
            max_diff = diff.max().item()
            assert max_diff < tol, f"Input {i}: max GPU/CPU diff = {max_diff} > {tol}"

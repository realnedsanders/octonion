    1. Create scripts/analyze_landscape.py following the pattern of analyze_stability.py:
       ```python
       """Phase 5: Optimization Landscape Analysis

       Reads experiment results from results/landscape/ and produces:
       - Go/no-go gate verdict (gate_verdict.json)
       - Convergence profile plots per task per optimizer
       - Hessian eigenspectrum evolution plots
       - Bill & Cox curvature comparison
       - Gradient variance characterization
       - Full structured report (full_report.json)
       - Pairwise statistical comparisons with Holm-Bonferroni correction

       Usage:
           docker compose run --rm dev uv run python scripts/analyze_landscape.py --results-dir results/landscape
       """
       ```

    2. Implement analysis functions:

       a. load_results(results_dir) -> dict:
          - Walk results_dir/{task}/{optimizer}/{algebra}/seed_{N}/result.json
          - Aggregate into nested dict: {task: {optimizer: {algebra: {seed: result}}}}
          - Report missing runs

       b. plot_convergence_profiles(results, output_dir):
          - For each task: one figure with subplots per optimizer
          - Each subplot: 6 lines (one per algebra), showing median val_loss across seeds
          - Error bands: 25th-75th percentile
          - Save as convergence_profiles_{task}.png

       c. plot_hessian_evolution(results, output_dir):
          - For each algebra: 4 panels (one per checkpoint)
          - Each panel: histogram of eigenvalue distribution
          - Overlay negative eigenvalue ratio as text
          - Save as hessian_spectra_{algebra}.png
          - Summary plot: negative eigenvalue ratio vs training progress for all algebras

       d. plot_curvature_comparison(results, output_dir):
          - Bar chart: mean curvature per algebra (error bars from std)
          - Grouped by task
          - Save as curvature_comparison.png

       e. plot_gradient_variance(results, output_dir):
          - For each task: gradient norm variance across seeds, plotted per algebra
          - Save as gradient_variance_{task}.png

       f. compute_pairwise_stats(results) -> dict:
          - For each task x optimizer: pairwise comparisons of best_val_loss across algebras
          - Apply holm_bonferroni correction
          - Report effect sizes (Cohen's d) and confidence intervals

       g. build_full_report(results, gate_result, pairwise_stats) -> dict:
          - Structured JSON with all results, gate verdict, statistical tests
          - Save as full_report.json

       h. Produce gate_verdict.json:
          - Call evaluate_gate with aggregated results
          - Save {verdict, per_task_ratios, divergence_rates, summary_text}

    3. Run the experiments:
       Before analysis, the experiments must actually run. The script should:
       - If results/landscape/ exists and has results, analyze existing
       - If --run flag: run experiments first via run_landscape_experiment, then analyze
       - Default: analyze-only mode (assumes experiments already ran)

    4. Literature comparison section:
       - Compare curvature values against Bill & Cox (2024) quaternion results (if available)
       - Compare convergence patterns against Wu et al. (2020) DON results
       - Document comparisons in full_report.json under "literature_comparison" key

    5. Handle the full experiment matrix:
       The full matrix is 6 algebras x 5 tasks x 5 optimizers x 20 seeds = 3000 runs.
       At ~5 min per run, this is ~250 GPU hours -- likely requires multiple sessions.
       The script should:
       - Report progress: "N/3000 runs complete, estimated remaining time"
       - Support partial analysis: analyze whatever results exist
       - Warn about missing (algebra, task, optimizer) combinations

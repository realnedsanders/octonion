    1. Install new dependencies inside container:
       `docker compose run --rm dev uv add geoopt pytorch-optimizer`

    2. Extend AlgebraType in _config.py with two new members:
       ```python
       PHM8 = ("PHM8", 8, 1)      # 8D, multiplier 1 (param matching handles A_i overhead)
       R8_DENSE = ("R8D", 8, 1)    # 8D, multiplier 1 (dense mixing has 64x params per feature pair)
       ```
       The multiplier=1 is nominal -- actual param counts differ due to layer structure.
       Parameter matching uses find_matched_width binary search, not multiplier.

    3. Create src/octonion/baselines/_phm_linear.py implementing PHM8Linear:
       - Based on Zhang et al. ICLR 2021 formulation: H = sum_{i=0}^{n-1} kron(A_i, S_i)
       - n=8 (matching octonionic dimension)
       - A_i: n learned (8x8) mixing matrices as nn.Parameter (n*n*n = 512 params)
       - S_i: n learned (out_features x in_features) sub-matrices as nn.Parameter
       - Forward: flatten x from [..., in_f, 8] to [..., in_f*8], compute H = sum_i kron(A_i, S_i), apply F.linear(x_flat, H), reshape to [..., out_f, 8]
       - Optional bias: Parameter of shape [out_features * 8], added in flat space then reshaped
       - Init: A_i with torch.randn * 0.1, S_i with xavier_normal_
       - IMPORTANT: Store A as a single Parameter [n, n, n] and S as [n, out_f, in_f] for cleaner parameter counting, or use nn.ParameterList. The total param count must be verifiable.

    4. Create src/octonion/baselines/_dense_mixing.py implementing DenseMixingLinear:
       - Full mixing: nn.Linear(in_features * dim, out_features * dim) equivalent
       - Weight: Parameter([out_f * 8, in_f * 8]), bias: Parameter([out_f * 8])
       - Forward: flatten x from [..., in_f, 8] to [..., in_f*8], F.linear, reshape to [..., out_f, 8]
       - Init: xavier_normal_ on weight
       - This is deliberately simple -- it's the "no algebra structure" baseline

    5. Update _param_matching.py:
       - Import PHM8Linear and DenseMixingLinear
       - In _SimpleAlgebraMLP.__init__, add dispatch for AlgebraType.PHM8 -> PHM8Linear and AlgebraType.R8_DENSE -> DenseMixingLinear
       - find_matched_width already uses binary search on _SimpleAlgebraMLP param count, so it will work automatically once the layer dispatch is added

    6. Update __init__.py:
       - Import and export PHM8Linear and DenseMixingLinear
       - Add to __all__

    7. After installing deps, run `docker compose run --rm dev uv sync` to ensure lockfile is consistent.

<objective>
Build the experiment orchestration infrastructure that ties together tasks, trainer, Hessian analysis, curvature measurement, and gradient variance collection into a single configurable pipeline with incremental result saving.

Purpose: The experiment matrix is massive (6 algebras x 9 tasks x 5 optimizers x 20 seeds). Orchestration must be robust to interruption and restartable. This plan does NOT run the full experiment -- it builds and smoke-tests the pipeline.
Output: Tested experiment runner with incremental saves, configurable via LandscapeConfig; a smoke test proves end-to-end validity.
</objective>

       ```python
       def run_landscape_experiment(config: LandscapeConfig) -> dict:
           """Run the full landscape experiment with incremental saving.

           Outer loop: task -> optimizer -> algebra -> seed
           At each (task, optimizer, algebra, seed):
             1. Check if result already exists in output_dir (skip if so)
             2. Build model, build data, train
             3. Save result JSON immediately after training
             4. If seed in hessian_seeds, save model checkpoints at hessian_checkpoints

           After all training:
             5. Run Hessian analysis on saved checkpoints
             6. Run curvature measurement on converged models
             7. Collect gradient variance statistics

           Returns: aggregated results dict
           """
       ```

       ```python
       def _result_path(output_dir: str, task: str, optimizer: str, algebra: str, seed: int) -> Path:

       def _result_exists(output_dir: str, task: str, optimizer: str, algebra: str, seed: int) -> bool:
           return _result_path(output_dir, task, optimizer, algebra, seed).exists()
       ```
       Each completed run saves a JSON with: train_losses, val_losses, best_val_loss, final_val_loss, epochs_trained, total_time_seconds.
       On resume, existing results are skipped.
